# Wendy Project Briefs

This repository collects concise project briefs for work that is not fully represented by a single public code repository yet. Each brief explains the problem, role scope, engineering or research signal, disclosure boundary, and what should not be overclaimed.

## Writing Principles

- Be specific about workflow, modules, constraints and validation thinking.
- Separate public code evidence, private/local project evidence and academic experience.
- Avoid claims about production traffic, commercial results, large-scale deployment or model performance unless there is public evidence.
- Keep private data, credentials, customer records and non-public research materials out of GitHub.
