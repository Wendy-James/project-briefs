# Disclosure Policy

This repository is intended for public project summaries only.

## Do Not Include

- Private source code
- Internal project documents
- Proprietary datasets
- Non-public prompts or model traces
- Confidential partner, client, or user information
- Unverified performance metrics
- Claims of deployment, publication, commercialization, or production use without evidence

## Recommended Language

Use:

- "This project explored..."
- "The system was designed to..."
- "The research focused on..."
- "The public brief does not include..."
- "Detailed quantitative results are not included because they are non-public or outside the scope of this public brief."

Avoid:

- "Open-sourced implementation" unless code is actually included
- "Achieved X% improvement" without public evidence
- "Deployed at scale" without verifiable support
- "Industry-leading" or similar unsupported claims

## Ownership And Confidentiality

When a project involves collaborators, school work, research programs, private data, or unpublished materials, the brief should describe only public and non-sensitive information. The goal is to communicate problem-solving ability and engineering judgment without exposing restricted material.
